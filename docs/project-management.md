# Gestión del Proyecto

Para la organización y desarrollo de MacroSolver AI, estamos utilizando un enfoque basado en **Kanban** mediante un tablero de **Trello**. Esto nos permite visualizar el flujo de trabajo y limitar el trabajo en progreso (WIP).

## Estructura del Tablero
El tablero está dividido en las siguientes columnas para reflejar el ciclo de vida de cada tarea:
1. **Backlog:** Funcionalidades futuras, ideas y tareas del servidor/IA que se abordarán en fases posteriores.
2. **Todo:** Tareas priorizadas para el Sprint actual (actualmente enfocado en el Frontend y la UI).
3. **In Progress:** Tareas que están siendo desarrolladas en este momento (máximo 2-3 tarjetas para evitar cuellos de botella).
4. **Review:** Tareas completadas que requieren revisión de código o pruebas manuales antes de darse por definitivas.
5. **Done:** Tareas terminadas, probadas y documentadas.

## Flujo de Trabajo
Cada funcionalidad principal (ej. "Sistema de Escáner") se divide en subtareas técnicas más pequeñas (ej. "Crear componente de cámara en React", "Configurar endpoint en Express"). Las tarjetas se mueven de izquierda a derecha a medida que avanza el desarrollo, asegurando un progreso constante y documentado.
